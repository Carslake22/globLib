1 # Introduction
2 
3 This library contains low level modules designed to interface with ATTINY13A perihperals.
4 These modules are designed to be compatible with other low level common libraries and
5 microcontroller modules as demonstrated on the [mainpage](https://github.com/stuianna/globLib).
6 
7 Development has been tested on Ubuntu 16.04LTS and 18.04LTS.
8 
9 # Installation
10 
11 ## Toolchain
12 
13 The [avr-gcc](https://gcc.gnu.org/wiki/avr-gcc) is used to devolop firmware for this device. It can be installed from the
14 package manager using:
15 
16 ```
17 sudo apt-get update
18 sudo apt-get install gcc-avr
19 ```
20 
21 Installation can be checked with
22 
23 ```
24 avr-gcc -v
25 ```
26 
27 Which should print something like this (or newer)
28 
29 ```
30 gcc version 4.9.2 (GCC)
31 ```
32 
33 Additionally, [avr-libc] (https://www.nongnu.org/avr-libc/) is required. It can be installed with:
34 
35 ```
36 sudo apt-get install avr-libc
37 ```

39 ## USBASP
40 
41 [USBASP](https://www.fischl.de/usbasp/) provides a hardware interface between the PC and MCU.
42 
43 USBASP has been found to work without any further configuration. It can be connected to the MCU as follows:
44 
45 |USBASP |ATTINY13A |
46 |---------------|:-----------------:|
47 |MOSI (PIN1) |MOSI (PIN 5) |
48 |RST (PIN 5) |RST (PIN 1) |
49 |SCK (PIN 7) |SCK (PIN 7) |
50 |MISO (PIN 9) |MISO (PIN 6) |
51 
52 The USBASP can provide power to the device (through pins 2 and 10) only when flashing firmware. This means that an external power supply (3.3V - 5V) should be used to
53 run the application.
54 
55 ## Dialout
56 
57 The user needs to be a member of group "dialout" to access the tty ports. This can be done with
58 
59 ```
60 sudo adduser $USER dialout
61 ```
