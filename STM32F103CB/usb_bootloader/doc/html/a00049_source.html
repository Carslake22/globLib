1 # STM32F103 USB Bootloader
2 
3 The USB bootloader is designed to upload new firmware on STM32F1 devices. It works
4 in conjunction with [dfu-util](http://dfu-util.sourceforge.net/) to send new programs to the microcontroller. The bootloader
5 jumps to the main application should none of the following conditions are met.
6 
7 - Holding Port A pin 1 high and cycle the power to the device.
8 - Sending 8 null characters ( \0 ) sequentially through the USB port.
9 - An occurrence of a hard fault
10 - A valid program not found at the application start address (0x08003000)
11 
12 The bootloader checks for these conditions using the first data register
13 in the backup domain. If this register is used for another function then the bootloader
14 may not function as expected.
15 
16 The majority of this bootloader is adapted from libopencm3 examples.
17 Modifications have been made to trigger the events above.
18 
19 The main application should be built with the [bootloader_project_template](www.). This
20 template incorporates a linker script to offset the application start address as well as
21 the USB peripheral setup.
22 
23 Complete installation instructions can be found in the STM32F103CB globLib main directory, or by typing **make libdoc** from this directory and navigating to STM32F103CB on the top tab bar.
24 
25 The following components are required by this template.
26 
27 - A local copy of [globLib](https://github.com/stuianna/globLib)
28 - arm-non-eabi-gcc
29 - STLink-V2 (To flash the original bootloader)
30 - OpenOCD
31 - Doxygen (Optional for document generation)

43 ## Usage
44 
45  1. Clone the repository.
46  2. Configure the makefile so the path to GlobLib is correct.
47  3. Connect the MCU and STLink.
48  4. Run **make all** to build the program.
49  5. Run **make flash** to flash the program to the MCU.
50  6. Done.
51 
52 Future programs can now be flashed to the MCU with the USB peripheral and the USB bootloader template program [link](www.google.com)

54 ## Make Recipes
55 
56 - **make** - Build project.
57 - **make clean** - Clean object and output files.
58 - **make flash** - Deploy program to microcontroller with STlink.
59 - **make erase** - Erase the flash memory of the STM32F1 (Removes any data and program).
60 - **make libdoc** - View the GlobLib Doxygen documentation
61 - **make docs** - Compile project Doxygen documentation (Requires Doxygen)
62 - **make viewdoc** - View USB bootloader specific documentation
63 - **make cleandoc** - Remove project specific documentation
