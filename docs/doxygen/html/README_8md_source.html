1 # Introduction
2 
3 GlobLib is combination of a hardware abstraction layer and common libraries which can be
4 shared across multiple microcontroller units (MCU). It is designed to provide boilerplate,
5 cross-MCU compatible code implementation of the most commonly used features. This project
6 was created to both support code reuseability and as an education exercise in multi-architecture firmware develop for myself.
7 
8 GlobLib focuses on proving functionality and portability for the most common features
9 of each peripheral.

51 # Installation / Usage
52 
53 This whole repository should be cloned onto a local machine to function correctly.
54 
55  1. Clone this repository onto the local machine
56  2. Run `chmod +x config.sh` to make the configuration file executable
57  3. Execute the config file with `./config.sh`. This script sets the glob_lib variable in each Makefile to the current root directory. This should also be run if the main glob_lib folder is moved.
58 
59 Each supported MCU has a templates containing a project template and specific
60 instructions for its usage. These are located in under each MCU name.
